# DQS の既定のナレッジ ベースの使用
  このトピックは、既定のナレッジ ベースをについて説明 **DQS データ**, 、と共にインストールされる [!INCLUDE[ssDQSnoversion](../includes/ssdqsnoversion-md.md)] (DQS)。 これは、次のドメインを含む、あらかじめ構築された既定のナレッジ ベースです。  
  
-   **国/地域**: 通常の長い (国/地域が指定した公式名) と短い名前 (一覧やマップなどで使用される共通名)、2 文字の省略形、3 文字の省略形、および場所ごとに 3 桁のコードが含まれています。  先頭の値は長い名前に設定されます。  
  
-   **国/地域 (3 文字が先頭)**: 通常の長い (国/地域が指定した公式名) と短い名前 (一覧やマップ、およびなどで使用される共通名)、2 文字の省略形、3 文字の省略形、および場所ごとに 3 桁のコードが含まれています。  先頭の値は、3 文字の省略形に設定されます。  
  
-   **国/地域 (2 文字が先頭)**: 通常の長い (国/地域が指定した公式名) と短い名前 (一覧やマップなどで使用される共通名)、2 文字の省略形、3 文字の省略形、および場所ごとに 3 桁のコードが含まれています。  先頭の値は、2 文字の省略形に設定されます。  
  
-   **米国 - 郡**: 米国の郡の一覧が含まれています。  
  
-   **米国 - 姓、名**: 100 発生している最後の名前 (姓) の何度も調査 2000年の一覧が含まれています。  
  
-   **米国 - 場所-**: 50 州、コロンビア特別区、および、国勢調査 2010年から抽出されたプエルトリコの地域の一覧が含まれています。  
  
-   **米国 - 州**: 通常の長い (公式) を含む名前と米国の各州の 2 文字の省略形です。 先頭の値は通常の州名に設定されます。  
  
-   **米国-州 (2 文字の見出し)**: 通常の長い (公式) を含む名前と米国の各州の 2 文字の省略形です。 先頭の値は、州名の 2 文字の省略形に設定されます。  
  
## 既定のナレッジ ベースの使用  
 既定の DQS ナレッジ ベースである DQS データは、次の方法で使用します。  
  
-   既定のナレッジ ベースを使用してクレンジング データ品質プロジェクトを開始して実行します。先に DQS の新しいナレッジ ベースを作成する必要はありません。  
  
-   既定のナレッジ ベースで、ドメイン管理、ナレッジ検出、または照合ポリシーのアクティビティを実行します。 これを行うには、次のようにクリックします。 **ナレッジ ベースを開く** で、 [データ品質クライアント ホームの画面](../data-quality-services/data-quality-client-home-screen.md), を選択、 **DQS データ** でナレッジ ベース、 **ナレッジ ベースを開く** 画面で、で必要なアクティビティをクリックして、 **アクティビティの選択** 領域です。 **[次へ]** をクリックします。  
  
-   既定のナレッジ ベースを使用して新しいナレッジ ベースを作成します。 既存のナレッジ ベースからナレッジ ベースを作成するを参照してください。 [ナレッジ ベースを作成](../data-quality-services/create-a-knowledge-base.md)します。  
  
-   使用する、 [Integration Services の DQS クレンジング コンポーネント](http://go.microsoft.com/fwlink/?LinkId=238830) と [マスター データ サービス アドイン Excel 用](../master-data-services/microsoft-excel-add-in/data-quality-matching-in-the-mds-add-in-for-excel.md)します。
