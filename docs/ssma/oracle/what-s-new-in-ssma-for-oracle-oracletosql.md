# <a name="what39s-new-in-ssma--for-oracle-oracletosql"></a>どのような &#39; SSMA for Oracle (OracleToSQL) の
このトピックでは、Oracle での変更の各リリース SSMA が一覧表示します。  

## <a name="may-2016"></a>2016 年 5 月  
SSMA for Oracle の 2016 年 5 月リリースには、次の変更が含まれています。  

1.  SQL Server 2016 のサポートが追加されました
2.  SQL Server のテンポラル テーブルにフラッシュ バック アーカイブ、Oracle テーブルの追加の変換
3.  SQL Server ポリシー オブジェクト (行レベルのセキュリティを Oracle) に変換する Oracle VPD ポリシーの追加の変換
4.  Oracle 用の初期読み込みの時間の削減
5.  強化されたパーサーとの競合回避モジュール
6.  .Net 2.0 のインストーラーのチェックを削除
7.  更新された拡張機能パックの依存関係から .Net 3.5 を .Net 4.0
8.  [保存] プロジェクト"固定と SSMA コンソールの「プロジェクトを開く」コマンド
9.  SSMA コンソールの固定"securepassword"コマンド
10. 初期読み込みのためのオブジェクトのカウントを修正
11. Oracle の文字データ型の変換を修正
12. グローバル設定のバグを修正
  
## <a name="march-2016"></a>2016 年 3 月  
SSMA for Oracle の 2016 年 3 月のプレビュー リリースには、次の変更が含まれています。  
  
1.  SQL Server 2016 への移行をサポートします。  
  
2.  移行する Oracle 行レベル セキュリティ (をいくつかの制限) のサポート  
  
3.  移行のサポートを SQL Server の列ストア テーブルをメモリ内での Oracle  
  
## <a name="january-2016"></a>2016 年 1 月  
2014 年 1 月 SSMA for Oracle のメンテナンスのリリースには、次の変更が含まれています。  
  
1.  クラスター化インデックスのサポートが追加されました  
  
2.  固定速度の遅い Oracle スキーマ クエリ (RFC 5076207)  
  
3.  固定コンソールから Azure への接続  
  
4.  SSMA (RFC 5706203) に追加された View ログ メニュー項目  
  
5.  追加された製品利用統計情報  
  
## <a name="july-2014"></a>2014 年 7 月  
SSMA for Oracle の 2014 年 7 月リリースには、次の変更が含まれています。  
  
1.  Azure SQL DB のサポート  
  
2.  Azure SQL DB をサポートするために、拡張機能パックの機能がスキーマに移動  
  
3.  Oracle が具体化されたビューのサポート  
  
4.  サポートの SQL Server 2014 メモリ最適化テーブル  
  
5.  10 k 個以上のデータベースのオブジェクトをテスト、パフォーマンスの向上  
  
6.  多数のオブジェクトを処理するための UI 機能強化  
  
7.  LOB スキーマの「既知」の強調表示  
  
8.  変換速度の向上  
  
9. UI 内のオブジェクトの数を表示します。  
  
10. 25% 以上のレポート サイズの縮小  
  
11. 未解析のコンス トラクターに対して改善されたエラー メッセージ。  
  
## <a name="april-2014"></a>2014 年 4 月

1.  MS SQL Server 2014 のサポートが追加されました。  
  
2.  Oracle 12 とクエリの最適化のサポートが追加されました。  
  
3.  Azure への変換に関する修正されたバグはします。  
  
4.  IE 10 で非表示レポート ページに関する修正されたバグはします。  
  
## <a name="january-2012"></a>2012 年 1 月  
SSMA for Oracle の 2012 年 1 月リリースには、次の変更が含まれています。  
  
-   サポートの RowType と RecordType の入力パラメーターの既定値は NULL です。  
  
## <a name="july-2011"></a>2011 年 7 月  
SSMA for Oracle の 2011 年 7 月リリースには、次の変更が含まれています。  
  
-   Oracle シーケンスへの変換のサポート[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]"Denali"のシーケンス ジェネレーター。  
  
-   エラー報告データの移行中の強化。  
  
-   予約語を使用してステートメントが強化された変換されます。  
  
-   関数の日付値の暗黙的な変換が向上します。  
  
## <a name="april-2011"></a>2011 年 4 月  
SSMA for Oracle の 2011 年 4 月リリースには、次の変更が含まれています。  
  
-   "Oracle SSMA"製品の統合をサポートする[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]2005 では、 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2008 と[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]"Denali"です。  
  
-   接続してへの移行のサポート[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]"Denali"です。  
  
-   クライアント側のデータ移行エンジン、データの並列移行をサポートする強化されています。  
  
-   強化されたデータの移行とパフォーマンスを単純な一括ログ復旧モデルに記録します。  
  
-   SSMA (v4.0 および v4.2) の以前のバージョンで作成されたプロジェクトの旧バージョンとの互換性。  
  
-   SSMA for Oracle v5.0 製品であることには、SSMA (v4.0 および v4.2) の旧バージョンとサイド バイ サイド (SxS) がインストールされています。  
  
-   ユーザー定義型 (サブタイプ、VARRAY、入れ子になったテーブル、テーブルのオブジェクトおよびオブジェクトのビューが含まれています) と特殊なエラー メッセージと共に PL/SQL ブロックでその使用状況をレポート作成をサポートします。

-   データの移行で"Custom SELECT"ステートメントのサポート  
  
-   Oracle 11 g R2 に移行するためのサポート  
  
## <a name="june-2008"></a>2008 年 6 月  
SSMA for Oracle の 2008 年 6 月リリースには、次の変更が含まれています。  
  
-   評価レポートの機能強化が完了しました。 シノニムの追加情報を解析可能なオブジェクト、パネルと SQL Server ロゴの削除、およびレイアウトの永続性の生データ ソースが含まれています。  
  
-   オブジェクトへの変換での改良:  
  
    -   パッケージ DBMS_LOB、DBMS_SQL 変換を追加します。  
  
    -   変換の変更を結合します。  
  
    -   コレクションとレコードの変換、フィールドごとに別々 の変数を使用してリリース単純の場合のみ、レコードの変換では今すぐ変更します。  
  
    -   レコードおよびコレクションの実装の機能を強化します。  
  
    -   ウィンドウ集計関数が追加されます。  
  
    -   プログラムのロールアップ/CUBE 句を追加します。  
  
    -   NEXTVAL/CURVAL の向上しています。  
  
    -   SET 句でグループ化、列セットをグループ化と ID をグループ化が追加されました。  
  
    -   MERGE ステートメントが追加されます。  
  
    -   新しい datetime 型およびレコードとコレクションの変換時に追加された CLR データ型のサポート。  
  
-   テスト担当者の新しい機能が追加されました。 テーブルを Tester を使用して、オブジェクトとしてテストできる、テスト ケース内のいくつかのテストが容易なオブジェクトの呼び出しの順序を変更することができます、ユーザーことができますプロシージャとレコードとコレクションをパラメーターとして関数をテスト値、および確認にアナライザーが追加された依存関係のみ使用されるテーブルを返します。  
  
## <a name="august-2007"></a>2007 年 8 月  
SSMA for Oracle の 2007 年 8 月リリースには、次の変更が含まれています。  
  
-   テスト担当者の新しいコンポーネントでは、作成、管理、および変換後の SQL コードを検証するテスト_ケースを実行することができます。  
  
-   SQL コンバーターには、Oracle のサブタイプ、コレクション、およびローカル モジュールの変換が追加されました。  
  
-   新しい同期機能では、SQL Server データベースと特定のオブジェクトを同期できます。  
  
-   新しい変換オプションを追加します。
