# <a name="project-settings-migration-mysqltosql"></a>プロジェクトの設定 (移行) (MySQLToSQL)
移行 ページ、**プロジェクト設定** ダイアログ ボックスには、どのように SSMA データが移行 MySQL から SQL Server をカスタマイズする設定が含まれています。  
  
移行ペインがで使用できる、**プロジェクト設定**と**プロジェクト設定の既定の** ダイアログ ボックス。  
  
-   すべての SSMA プロジェクトの設定を指定する、**ツール**メニューの [**プロジェクト設定の既定の**、プロジェクトの種類を選択**移行対象のバージョン**ドロップ ダウン] をクリックするとなる設定にアクセスし、**全般**クリックして、左側のウィンドウの下部にある**移行**です。  
  
-   現在のプロジェクトの設定を指定する、**ツール**メニューの [**プロジェクト設定**、] をクリックして**全般**クリックして、左側のウィンドウの下部にある**移行**です。  
  
## <a name="options"></a>オプション  
  
### <a name="bulk-copy"></a>一括コピー  
  
|項目|定義|  
|--------|--------------|  
|**バッチ サイズ**|バッチの指定サイズのデータの移行時に使用します。<br /><br />**既定のモード**: 1000<br /><br />**オプティミスティック モード**: 1000<br /><br />**Full モード**: 1000|  
|**CHECK 制約**|これを SQL Server テーブルにデータを挿入するときに、SSMA が制約をチェックするかどうかを指定します。<br /><br />**既定のモード**: False<br /><br />**オプティミスティック モード**: False<br /><br />**Full モード**: False|  
|**[トリガーを起動する]**|SQL Server テーブルにデータを追加する場合に、SSMA が挿入トリガーを起動するかどうかを指定します。<br /><br />**既定のモード**: False<br /><br />**オプティミスティック モード**: False<br /><br />**Full モード**: False|  
|**[ID を保持する]**|SSMA では、SQL Server にデータを追加する場合に、MySQL id 値が保持されるかどうかを指定します。 値 False の場合、変換先で割り当てられる id 値です。<br /><br />**既定のモード**: True<br /><br />**オプティミスティック モード**: True<br /><br />**Full モード**: True|  
|**[NULL を保持する]**|SSMA では、SQL Server で指定されている既定値に関係なく、SQL Server にデータを追加する場合に、ソース データ内の null 値が保持されるかどうかを指定します。<br /><br />**既定のモード**: True<br /><br />**オプティミスティック モード**: True<br /><br />**Full モード**: True|  
|**[テーブル ロック]**|SSMA がデータの移行中のテーブルにデータを追加する場合に、テーブルをロックするかどうかを指定します。 一括コピー操作の間で一括更新ロックを取得します。 値が False の場合は、ロックは行レベルで設定されます。<br /><br />**既定のモード**: False<br /><br />**オプティミスティック モード**: False<br /><br />**Full モード**: False|

|**負の時間値の移行**|同様に負の値を移行する方法を指定します '-30:11:00' 時刻列にします。<br /><br />**既定のモード**: NULL に設定<br /><br />**オプティミスティック モード**: NULL に設定<br /><br />**Full モード**: NULL に設定|  
|**時刻値 24 時間以上の移行**|時刻値の複数の ' 23 を移行する方法を指定します: 59:59' 時刻列にします。<br /><br />**既定のモード**: NULL に設定<br /><br />**オプティミスティック モード**: NULL に設定<br /><br />**Full モード**: NULL に設定|  
|**列に収まるようにバイナリ値を切り捨てる**|場合は、SSMA は MySQL から SQL テーブルの列に適合しないバイナリの値を切り捨てます。 し、適切なエラー メッセージを生成します。 × の場合、行、エラーを発生します。<br /><br />**既定のモード**: いいえ<br /><br />**オプティミスティック モード**: いいえ<br /><br />**Full モード**: いいえ|  
|**列に収まるように文字値を切り捨てる**|SSMA は、MySQL から SQL テーブルの列に適合しない文字の値を切り捨てます。 し、適切なエラー メッセージを生成します。<br /><br />**既定のモード**: いいえ<br /><br />**オプティミスティック モード**: いいえ<br /><br />**Full モード**: いいえ|  
|**0 の日付の移行**|ように 0 の日付を移行する方法を指定します '0000-00-00' または' 0000-00-00 00時 00分: 00' DATE および datetime 型の列にします。<br /><br />**既定のモード**: NULL に設定<br /><br />**オプティミスティック モード**: NULL に設定<br /><br />**Full モード**: NULL に設定|  
|**日付の移行では 0 します。**|ように 0 個の部分と日付を移行する方法を示す '2009-01-00' または' 2000-00-00 11時 00分: 00' DATE および datetime 型の列にします。<br /><br />**既定のモード**: NULL に設定<br /><br />**オプティミスティック モード**: NULL に設定<br /><br />**Full モード**: NULL に設定|  
  
### <a name="migration-engine"></a>移行エンジン  
  
|項目|定義|  
|--------|--------------|  
|**移行エンジン**|指定データベース エンジンがデータの移行時に使用します。 クライアント側のデータ移行を指す SSMA クライアントから、ソースとデータの一括挿入する SQL Server にデータを取得します。 サーバー側のデータの移行は、SQL Server のボックスで、ソースからデータを取得して、余分なクライアントのホップ (パフォーマンスの向上) を回避する SQL Server に直接挿入する SQL エージェント ジョブとして実行する SSMA データ移行エンジン (一括コピー プログラム) を参照します。<br /><br />**既定のモード**: クライアント側のデータ移行のエンジン<br /><br />**オプティミスティック モード**: クライアント側のデータ移行のエンジン<br /><br />**Full モード**: クライアント側のデータ移行のエンジン|  
  
> [!IMPORTANT]  
> ときに、**移行エンジン**にオプションが設定されている**サーバー側のデータ移行のエンジン**、オプションを設定する新しいプロジェクト**を使用して 32 ビット サーバー側のデータ移行エンジン**が表示されます。 これは、データを移行する 32 ビットまたは 64 ビットの一括コピー プログラム (BCP) ユーティリティを使用するかどうかを指定します。

### <a name="parallel-data-migration"></a>並列データの移行  
  
|項目|定義|  
|--------|--------------|  
|**データへの並列移行モード**|データへの並列移行を有効にする分岐のスレッドを使用するモードを指定します。 Auto モードでは、SSMA は、スレッドの数 (既定では 10) データを移行するために分岐を選択します。 カスタム モードでは、ユーザーがデータを移行するために分岐するスレッドの数を指定できます (最小値は 1 と最大値は 100)。 現時点では、クライアント側のデータ移行のエンジンだけでは、データへの並列移行をサポートしています。<br /><br />**既定のモード**: 自動<br /><br />**オプティミスティック モード**: 自動<br /><br />**Full モード**: 自動|  
  
> [!IMPORTANT]  
> ときに、**並列のデータ移行モード**にオプションが設定されている**カスタム**、オプションを設定する新しいプロジェクト**スレッド数**が表示されます。 データの移行に使用されるスレッドの数を指定します。  
  
### <a name="spatial-data"></a>空間データ  
  
|項目|定義|  
|--------|--------------|  
|**エラーの処理**|空間データ型の値の移行でエラーを処理する方法を指定します。 'Replace では NULL' が指定されている場合は、NULL とすべての空間値が原因で、エラーに交換してください。 置換は行われませんそれ以外の場合。<br /><br />**既定のモード**: エラーが発生<br /><br />**オプティミスティック モード**: エラーが発生<br /><br />**Full モード**: エラーが発生|  
|**値の検証**|無効な空間の値を処理する方法を指定します。 'を再試行してください、有効な' が指定されている場合、しようとしましたが中に有効にするために無効な値を変更します。<br /><br />**既定のモード**: 有効にします。<br /><br />**オプティミスティック モード**: 変更しないでください<br /><br />**Full モード**: 有効にします。|
