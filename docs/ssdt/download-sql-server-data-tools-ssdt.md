# <a name="download-sql-server-data-tools-ssdt"></a>SQL Server Data Tools (SSDT) のダウンロード
無料でダウンロードできる最新の開発ツールである [!INCLUDE[appliesto-ss-asdb-asdw-pdw-md.md](../includes/appliesto-ss-asdb-asdw-pdw-md.md)]
**[SQL Server Data Tools](https://msdn.microsoft.com/library/hh272686(v=vs.103).aspx)** を使用すると、SQL Server リレーショナル データベース、Azure SQL Database、Integration Services パッケージ、Analysis Services データ モデル、Reporting Services レポートを作成できます。 SSDT では、Visual Studio でアプリケーションを開発する場合と同じくらい簡単に、SQL Server のコンテンツの種類を設計および展開できます。 

SSDT for Visual Studio 2017 (15.5.1) はプレビューが終了し、一般公開 (GA) リリースになりました。 Visual Studio 2017 (15.5 以降) の SQL Server データベース、Analysis Services、Reporting Services、Integration Services プロジェクトの場合、このリリースはスタンドアロン Web インストールになります。

> [!IMPORTANT]
> Visual Studio 2017 (15.5.1) はバージョン 15.5.0 と同じリリースですが、インストーラーのバグがいくつか修正されています。 基本的に同じリリースであるため、15.5.0 から 15.5.1 にアップグレードしないでください。 Visual Studio 2017 (15.5.0) を既にインストールしている場合、インストーラーの問題が解消されているので、15.5.1 に移行する理由はありません。 
> 
> SSDT for Visual Studio 2017 (15.5.1) をインストールする前に、"Microsoft Analysis Services プロジェクト" と "Microsoft Reporting Services プロジェクト" 拡張機能を VS 2017 に既にインストールしてある場合はアンインストールし、すべての VS インスタンスを閉じます。 
> 
> SSDT for Visual Studio 2017 15.5.1 の英語以外のバージョンでは、英語の 15.4.0 プレビューからのアップグレードはできません。他の言語で 15.5.1 をインストールするには、英語の 15.4.0 プレビューを先にアンインストールする必要があります。 


SSDT for Visual Studio 2015 と SSDT for Visual Studio 2017 は両方とも DacFx 17.4 を使用: [Data-Tier Application Framework (DacFx) 17.4 をダウンロードする](https://www.microsoft.com/download/details.aspx?id=56356)

### <a name="iso-images---ssdt-for-vs-2015"></a>ISO イメージ - SSDT for VS 2015

## <a name="installing-ssdt-without-visual-studio-pre-installed"></a>Visual Studio の事前インストールがない状態での SSDT のインストール

コンピューターに Visual Studio がインストールされていない場合、SSDT for Visual Studio をインストールすると、最小バージョンの Visual Studio がインストールされます。 このバージョンの Visual Studio は無料でインストールでき、必要な数のコンピューターで使用できます。 すべての種類の SQL Server プロジェクトに加え、*SQL Server オブジェクト エクスプローラー*および他の SQL ツールを使用できます。

Visual Studio 2015 (以降) が既にインストールされている場合、SSDT をインストールすると、既存の Visual Studio に完全な SQL Server ツール セットが追加されます。 Visual Studio には、ソース コード管理の統合および非 SQL 言語のサポートなど、使用可能な多くの機能が含まれています。 T-SQL の開発時に快適に操作できるように、Visual Studio 2015 以降を使用することをお勧めします。
